# Obsidian 阿里云盘同步插件使用指南

## 这是什么

这个插件把当前 Obsidian vault 同步到阿里云盘 App 里的一个云端文件夹。

这里的“阿里云盘文件夹”不是本机文件夹，也不是本机阿里云盘客户端同步出来的目录，而是你的个人阿里云盘账号中的云端目录。

多设备同步方式是：

1. 每台设备都安装这个插件。
2. 每台设备都使用同一个阿里云盘账号的 `refresh_token` 连接。
3. 每台设备都填写同一个云端同步文件夹路径。
4. 设备 A 修改笔记后上传到云端。
5. 设备 B 打开 Obsidian 后，或定时检查时，检测到云端更新并自动拉取。

## 当前授权方式

阿里云盘个人开发者申请已经暂停，所以当前插件主推“OpenList refresh_token 模式”：

1. 通过 OpenList Token 获取工具扫码登录阿里云盘。
2. 复制工具返回的 `Refresh Token`。
3. 回到插件设置页粘贴 `Refresh Token`。
4. 插件通过 OpenList 续期接口换取访问令牌，然后继续调用阿里云盘 Open API 同步文件。

注意：`refresh_token` 相当于长期登录凭证。插件只保存到本地，不会写入云端元数据，也不会写入日志；但你仍然不要把它发给别人。

## 安装到 Obsidian

1. 在项目根目录执行：

```bash
npm install
npm run build
```

2. 找到你的 Obsidian vault 插件目录：

```text
<你的 vault>/.obsidian/plugins/obsidian-aliyun-sync/
```

3. 把以下文件复制进去：

```text
main.js
manifest.json
styles.css
```

4. 打开 Obsidian。
5. 进入 `设置 -> 第三方插件`。
6. 关闭安全模式后启用 `Aliyun Drive Sync`。

## 获取 refresh_token

1. 打开 [OpenList Token 获取工具](https://api.oplist.org/)。
2. 在“网盘名称”里选择 `阿里云盘（OAuth2）扫码登录`。
3. 勾选 `使用 OpenList 提供的参数`。
4. `客户端ID` 和 `应用秘钥` 保持为空。
5. 点击 `获取 Token`。
6. 用阿里云盘 App 扫码登录并授权。
7. 页面显示 `访问令牌（Access Token）` 和 `刷新令牌（Refresh Token）` 后，复制 `刷新令牌（Refresh Token）`。

如果你担心第三方续期服务，可以后续自建 OpenList APIPages，然后在插件里把“OpenList 续期接口”改成自己的地址。

## 首次配置

进入 Obsidian 插件设置页：`设置 -> 第三方插件 -> Aliyun Drive Sync`。

建议配置：

1. `阿里云盘 Open refresh_token`
   - 粘贴上一步复制的 `Refresh Token`。

2. `获取 refresh_token`
   - 如果还没有 token，可以直接从这里打开 OpenList Token 获取工具。

3. `连接阿里云盘`
   - 粘贴或更新 token 后，点击连接刷新登录状态。

4. `云端同步文件夹`
   - 例如：

```text
/Apps/ObsidianSync/MyVault
```

   - 多台设备必须填写同一个路径。
   - 这个路径是阿里云盘云端路径，不是本地路径。

5. `设备名称`
   - 例如：

```text
Windows-PC
MacBook
iPhone
```

   - 设备名称会用于冲突文件命名。

6. `同步 .obsidian 配置目录`
   - 这个开关现在放在 `进阶配置` 中。
   - 默认关闭，建议先同步笔记内容，确认稳定后再开启。

`进阶配置` 默认折叠。里面包含 OpenList 续期接口、OpenList 授权类型、启动后同步延迟、自动同步间隔、保存后同步延迟、并行传输数量、删除保护阈值等设置。大多数情况下保持默认即可。

## 连接阿里云盘

1. 在设置页粘贴 `refresh_token`。
2. 点击 `连接`。
3. 插件会通过续期接口换取访问令牌。
4. 看到“阿里云盘连接成功”后，点击 `测试连接`。
5. 测试通过后，再点击 `立即同步`。

如果连接失败：

- 确认复制的是 `Refresh Token`，不是 `Access Token`。
- 确认 OpenList 工具里选择的是阿里云盘 OAuth2 扫码登录。
- 确认插件里的授权类型是 `alicloud_qr`。
- 如果提示 Too Many Requests，等一段时间再试，或换自建续期接口。
- 如果你在阿里云盘 App 中撤销了 OpenList 授权，需要重新扫码获取新的 token。

## 手动同步

你可以通过两种方式手动同步：

1. 设置页点击 `立即同步`。
2. 命令面板执行 `同步到阿里云盘`。

手动同步会：

1. 扫描本地 vault。
2. 扫描云端同步文件夹。
3. 读取本设备上一轮同步日志。
4. 生成同步计划。
5. 执行上传、下载、合并或冲突副本。
6. 更新同步日志。

同步过程中，Obsidian 底部状态栏会显示大体进度，例如：

```text
正在同步 2/5 - 扫描本地和云端文件
正在同步 8/42 - 上传: Notes/example.md
正在同步 4/5 - 写入同步记录
```

这个进度用于判断当前阶段和剩余同步项数量，不代表精确耗时。

## 同步速度和并行传输

插件支持并行上传和下载，但阿里云盘 Open API 容易受接口频率、下载并发和服务器负载影响，所以不建议把并行数开得太高。

建议：

- 默认保持 `3`。
- 网络稳定、文件较多时可以尝试 `4` 或 `5`。
- 如果出现 429、403、Too Many Requests、上传分片失败或下载失败，先降到 `2`。
- 如果仍然失败，降到 `1`，让本轮同步更稳。

插件会对疑似限流或网络抖动的上传、下载做重试。如果某一批并行传输失败，插件会在本轮同步中自动降低并行数后重试失败项；如果降到 `1` 仍然失败，本轮同步会停止，下一次同步会继续按增量状态重新判断。

## 自动同步

插件支持三种自动同步触发：

1. 启动后自动检查。
2. 定时检查。
3. 本地文件保存后延迟同步。

默认行为：

- 启动后约 10 秒检查一次。
- 定时同步间隔默认为 10 分钟。
- 保存后约 20 秒触发同步。
- 并行传输数量默认为 3。

你可以在 `进阶配置 -> 启动后同步延迟（秒）` 中调整打开 Obsidian 后多久触发第一次同步。设置为 `0` 表示打开后立即触发；关闭 `启动后自动检查` 则不会在打开时自动同步。

自动同步不是实时协同：

- Obsidian 关闭时，插件不会在后台运行。
- 另一台设备需要打开 Obsidian，才能检测云端更新。
- 检测依赖定时轮询，不依赖云端实时推送。

## 多设备使用方式

以两台设备为例：

### 设备 A

1. 安装插件。
2. 粘贴 `refresh_token` 并连接阿里云盘。
3. 设置云端同步文件夹：

```text
/Apps/ObsidianSync/MyVault
```

4. 点击 `立即同步`，把本地 vault 上传到云端。

### 设备 B

1. 安装插件。
2. 使用同一个阿里云盘账号重新获取或粘贴可用的 `refresh_token`。
3. 填写同一个云端同步文件夹：

```text
/Apps/ObsidianSync/MyVault
```

4. 点击 `立即同步`，或等待启动自动检查。
5. 插件会检测云端已有内容，并下载到本地 vault。

之后：

- A 修改并同步后，B 会在下一次自动检查时拉取。
- B 修改并同步后，A 会在下一次自动检查时拉取。

## 冲突处理

如果两台设备都修改了同一个文件：

- markdown 文件会尝试自动合并。
- 如果不能安全合并，会保留冲突标记。
- 二进制文件或无法合并的文件，会把冲突副本保存到插件内部目录：

```text
.obsidian-aliyun-sync/conflicts/
```

插件会默认忽略以下冲突副本命名，不再把它们当作普通笔记反复同步：

```text
*.conflict.*
**/*.conflict.*
*.sync-conflict-*
**/*.sync-conflict-*
```

如果你之前已经在云端目录里生成了很多 `.conflict.` 文件，升级后它们不会再被拉回本地。你可以在阿里云盘 App 里确认内容无用后手动删除它们。

默认策略是保守的：尽量自动合并 markdown；真正无法判断时才保留副本，而且不再污染正常笔记目录。

## 删除保护

删除同步默认开启，但有保护阈值。

如果一次同步将删除太多文件，插件会停止同步并提示失败，避免因为误配云端目录或误删导致大面积数据丢失。

建议：

- 首次同步前备份 vault。
- 不要把云端同步文件夹设为阿里云盘根目录。
- 不要让多个不同 vault 共用同一个云端同步文件夹。

## 常见问题

### 是否还需要申请阿里云盘开发者 ID？

当前方案不需要。你通过 OpenList 工具授权，插件只需要你粘贴 `refresh_token`。

### 是否需要安装阿里云盘客户端？

不需要。插件直接访问阿里云盘 Open API。

### 是否同步到本地阿里云盘目录？

不是。插件同步到阿里云盘云端文件夹。

### 另一台设备会立即收到更新吗？

不会保证“立即”。另一台设备需要打开 Obsidian，并在启动检查、定时轮询或手动同步时拉取更新。

### token 安全吗？

`refresh_token` 保存在 Obsidian 插件本地数据中，不会写入云端元数据，也不会写入日志。但它会被发送到你配置的 OpenList 续期接口，用来换取新的访问令牌。更高安全要求下，建议自建 OpenList APIPages。
